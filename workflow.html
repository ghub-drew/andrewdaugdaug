Framework Breakdown
01 · Trigger
When does it run?
The event or schedule that starts the agent.
Schedule-Based
📅
Every Monday morning before class
Sends weekly lesson plan and objectives to students
📋
End of each lesson (Friday)
Generates a recap summary and practice exercises
Event-Based
📤
Student submits HTML exercise
Agent reviews code and gives instant feedback
⚠️
Student scores below passing
Triggers extra practice set and a review tip
Custom Trigger
💬
Teacher types a command
"Generate quiz for HTML forms" runs instantly
02 · Context
What does it know?
The knowledge and tools the agent works with.
Knowledge Sources
📚
Course syllabus and topic order
HTML basics, tags, links, forms, tables, CSS intro
🗂️
Student progress records
Who is on which lesson, weak areas, past scores
🐛
Common beginner HTML mistakes
Unclosed tags, missing alt text, wrong nesting
Available Tools
✏️
Generate coding exercises by topic
Tags, forms, tables, images, links, semantic HTML
🔍
Check student HTML for errors
Flags issues with hints, not just the answer
📝
Create quizzes with answer keys
Multiple choice, fill-in, and short code tasks
03 · Steering
How do you keep it honest?
Controls that keep the teacher in charge.
Human Oversight
👀
Teacher reviews before sending
All exercises and quizzes go to teacher first
✅
Approve or edit outputs
Nothing reaches students without approval
Verification
🧪
Checks if generated HTML is valid
Agent tests its own output before showing it
📊
Validates quiz answers are correct
Confirms answer key before distributing
Operational Controls
⏸️
Pause during exams or holidays
Teacher controls when the routine is active
↩️
Redirect to new topic anytime
Syllabus moves forward, agent follows
Live Scenario · HTML Forms Lesson
🏫
Computer Teacher · Web Programming Class
Topic: HTML Forms · Grade 10 · 25 students
1
Trigger · Schedule
Sunday 8PM · Lesson prep runs automatically
The agent detects that Monday's topic is HTML Forms. It checks the syllabus, sees which students completed last week's links exercise, and prepares for this week's lesson.
2
Context · Knowledge
Agent loads what it knows about HTML Forms
Reads the syllabus entry for forms, pulls common beginner mistakes (missing name attributes, wrong input types, no labels), and checks which 3 students scored below 70% last week and need extra attention.
3
Context · Tools
Generates lesson materials for the teacher
Creates a 10-question quiz on HTML Forms with answer key, a coding exercise (build a contact form), and a one-page lesson summary the teacher can project in class.
4
Steering · Oversight
Teacher reviews everything before Monday
Teacher opens the draft, adjusts 2 quiz questions, approves the coding exercise as-is. Nothing is shared with students yet. Teacher is still in full control.
5
Trigger · Event
Student submits their HTML form exercise
A student uploads their contact form code. The agent is triggered instantly. It reads the HTML, checks for errors, validates structure, and prepares feedback.
6
Steering · Verification
Agent checks its feedback before sending
Confirms the feedback is accurate, not too harsh, and gives hints rather than full answers. The teacher gets a summary of all submissions with scores and flags for manual review.
7
Output · Autonomous Value
Teacher saved hours. Students got faster feedback.
The teacher spent 10 minutes reviewing instead of 3 hours creating. Each student got personalized feedback within minutes of submitting, not the next class day.
Sample Prompt Sent to Claude
# Role You are a helpful assistant for a computer teacher. Topic: HTML Forms · Grade 10 Web Programming Task: Generate a 10-question quiz and a coding exercise. # Context - Previous topic: HTML Links and Anchors (completed) - Students to watch: 3 students scored below 70% last week - Common mistakes: missing name attributes, no labels, wrong input types # Output Format 1. Quiz: 10 questions, mix of multiple choice and fill-in 2. Exercise: Build a contact form with name, email, message, submit 3. Answer key: included, not shown to students Tone: Clear, beginner-friendly, no jargon
